# Tommma

## Идея приложения
Tommma - это минималистичный визуальный таск-менеджер с фокусом на дне и времени в задаче.

Основная идея:
- быстро создавать задачи прямо в нужной колонке (`Todo` или конкретный день);
- видеть план времени по задаче;
- запускать таймер для задач текущего дня и накапливать факт;
- управлять неделей с шагом в день или неделю без перегруженного интерфейса.

Приложение сделано как personal-tool в браузере: локально работает через `localStorage`, а на хостинге поддерживает аккаунты с авторизацией по email и синхронизацией состояния в MySQL.

## Функционал v1
- Колонки: общий `Todo` + колонки дней недели (неделя начинается с воскресенья).
- Создание задачи в конкретной колонке:
  - inline-поле на месте кнопки `Добавить задачу`;
  - `Enter` - создать, `Esc` - отменить;
  - пустое название не создается.
- Быстрый ввод плана времени в названии:
  - формат `Название / 240`;
  - поддерживаются только целые минуты `1..1440`;
  - невалидный суффикс или `0` игнорируется (задача создается без плана).
- Редактирование задачи inline (двойной клик по названию) с тем же синтаксисом `/ минуты`.
- Чекбокс выполнения задачи (выполненная задача остается на том же месте).
- Удаление задачи с `Undo` на 5 секунд.
- Активную (запущенную) задачу удалять нельзя.
- Таймер:
  - `Play / Pause / Stop`;
  - одновременно может быть только один running-таймер;
  - запуск новой задачи автоматически ставит предыдущую running-задачу на паузу;
  - `Stop` добавляет накопленное время в факт;
  - таймер доступен только для задач текущего дня (для остальных - silent no-op).
- Авто-стоп при смене локальной даты устройства:
  - активный таймер останавливается,
  - накопленное время сохраняется в факт.
- Плеер снизу:
  - показывается при выборе задачи с планом;
  - у задачи появляется активная подсветка;
  - у задач без плана плеер не показывается.
- Отображение времени:
  - в плеере - `HH:MM:SS`;
  - в прогрессе - `mm`;
  - пока факта нет, показывается только план;
  - после появления факта (`> 0`) - `факт / план`.
- Навигация:
  - одна стрелка - сдвиг на 1 день;
  - двойная стрелка - сдвиг на неделю;
  - кнопка слева у панели - открыть/скрыть `Todo`.
- При скрытии `Todo` дневные колонки сдвигаются влево без изменения ширины.
- Мобильный режим:
  - одна колонка на экране;
  - свайп влево/вправо по одному дню;
  - `Todo` открывается кнопкой в модалке.

## Запуск
Откройте `index.html` в браузере.

## Подключение к базе Beget (MySQL)
В проект добавлен API для хранения состояния в MySQL:
- `api/state.php` - чтение/сохранение состояния приложения;
- `api/auth.php` - регистрация, вход, подтверждение email-кодом, выход;
- `api/db.php` - подключение к БД;
- `api/config.php` - конфиг доступа;
- `database.sql` - SQL-схема таблиц `users` и `app_state`.

### 1) Создайте таблицу
В phpMyAdmin на Beget выполните SQL из файла `database.sql`.
Если у вас уже была старая версия проекта без аккаунтов, `database.sql` пересоздаст `app_state` под новую схему (старое общее состояние будет удалено).

### 2) Укажите доступ к БД
Откройте `api/config.php` и заполните:
- `db_host` (обычно `localhost`);
- `db_name`;
- `db_user`;
- `db_pass`.

И заполните настройки отправителя почты:
- `auth@tommma.ru` (существующий email вашего домена);
- `Tommma` (например, `Tommma`);
- `auth@tommma.ru` (обычно тот же адрес, что `mail_from`).

### 3) Загрузите проект на хостинг
Загрузите файлы в корень сайта (где работает PHP), затем откройте сайт по домену.

### Как это работает
- Локально (при открытии файла как `file://`) приложение работает как раньше через `localStorage` без авторизации.
- На хостинге (`http/https`) приложение:
  - требует регистрацию/вход;
  - после регистрации отправляет 6-значный код на email и просит подтвердить почту;
  - при старте делает `GET /api/state.php` для текущего пользователя;
  - при любом изменении делает `POST /api/state.php` и сохраняет состояние по `user_id`.
- На локальном сервере (`localhost`, `127.0.0.1`, `::1`) доступен тестовый вход:
  - логин: `testuser`
  - пароль: `Test12345`
  - при работающем PHP API тестовый пользователь создается/обновляется автоматически при первом входе;
  - если проект запущен как статический сервер без PHP, используется локальный fallback-вход (без БД).

## Важно про отправку email
- Код подтверждения отправляется через PHP `mail()` из `api/auth.php`.
- На хостинге должна быть настроена отправка почты (обычно на Beget это уже доступно).
