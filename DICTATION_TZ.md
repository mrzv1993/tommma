# ТЗ: Диктовка и Распределение Задач

## Назначение
Функционал должен преобразовывать свободную речь в список задач с датой и группой, без хвостов и служебных слов, с обязательным предпросмотром и подтверждением.

## Обязательные правила
1. Смешанный формат входа: одна длинная фраза и список одинаково поддерживаются.
2. Текст проходит через LLM-нормализацию (пунктуация + readability).
3. Группа обязательна всегда:
   - по умолчанию `todo`;
   - при маркерах переопределяется на `not-do` или `anti-todo`.
4. Даты поддерживаются:
   - `12 марта`, `на 12 марта`;
   - `12.03`, `12/03`, `YYYY-MM-DD`;
   - `сегодня`, `завтра`, `послезавтра`.
5. Если год не указан, выбирать ближайшую будущую дату.
6. Время (`в 15:30`, `утром`) игнорируется.
7. Очистка хвостов обязательна:
   - служебные префиксы (`поставь задачу`, `запиши`, `добавь`, `todo`, `на туду` и т.д.);
   - остатки дат в названии;
   - мусорные связки на краях (`и`, `а`) и лишняя пунктуация.
8. Валидация задачи:
   - минимум 5 символов;
   - минимум 2 слова;
   - не входит в стоп-список `drop_exact`.
9. Дубликаты автоматически схлопываются по ключу `group + date + normalized title`.
10. Добавление в реальные колонки только после подтверждения в preview.

## Preview UI
Порядок полей в строке:
- `Группа (селект)` → `Название` → `Дата` → `Кнопки`.

Кнопки:
- для строки: `Подтвердить`, `Удалить`;
- для списка: `Подтвердить все`, `Очистить всё`.

## Настраиваемые правила
Файл: `dictation_rules.json`.

В этот файл можно добавлять:
- новые стоп-префиксы (`stop_prefixes`);
- новые маркеры групп (`group_markers`);
- хвостовые слова (`tail_words`);
- стоп-фразы для полного удаления (`drop_exact`).

Изменения в `dictation_rules.json` применяются сервером при следующем запросе диктовки.
